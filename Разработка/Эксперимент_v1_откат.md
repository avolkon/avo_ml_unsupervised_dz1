## 3. Проведённые эксперименты и LB

| # | Подход | Файл | Public LB | vs v0 |
|---|--------|------|-----------|-------|
| 0 | **v0 baseline** | `submission_pca_kmeans.csv` | **0.22835** | — |
| 1 | v1 row: subject-wise scaler + MiniBatchKMeans k=6 | `submission_v1.csv` (первая версия) | **0.11517** | −49% |
| 2 | v1 segment pipeline + majority vote k=10 | `submission_v1_segments.csv` | не отправлялся | — |
| 3 | Tuning: motion24 (Acc16+Gyro), k=7, PCA 95% | `submission_v1_improved.csv` | **0.16535** | −28% |

### 3.1. Почему v1 (subject-wise scaling) провалился

EDA рекомендовал нормализацию **внутри subject_id**, но на LB это дало **0.115** вместо **0.228**.

**Гипотеза:** тестовая выборка Kaggle, вероятно, оценивается **глобально по строкам**; subject-wise scaling меняет геометрию пространства признаков так, что кластеры перестают совпадать с разметкой платформы.

### 3.2. Почему «улучшенный» кандидат (k=7, motion24) хуже v0

| | v0 k=6 | improved k=7 |
|--|--------|--------------|
| Признаки | 42 (все сенсоры) | 18 (только Acc16+Gyro) |
| Крупнейший кластер | 78.4% | 73.6% |
| LB | 0.228 | 0.165 |

**Вывод:** более «сбалансированное» распределение кластеров и k=7 **не коррелируют** с accuracy на Kaggle. Доминирующий кластер ~78% в v0, видимо, ближе к реальной структуре теста.

### 3.3. Что не сработало (внутренние прогоны, без LB)

| Идея | Результат |
|------|-----------|
| Temporal median smoothing (w=21–101) | один кластер ~99% строк |
| Segment + majority vote | высокий internal score, но не v0-паттерн |
| StandardScaler + PCA 98% + MiniBatchKMeans | LB ниже v0 в v0-ноутбуке (0.147) |
| GMM на 534k строк | слишком медленно, не дало gain |

## 5. Воспроизведение v0 (после отката)

Логика совпадает с ячейкой «KMeans с PCA» в `avo_m25_555_ml_unsupervised_v0.ipynb`:

```python
# df_reduced: interpolate → drop Acc6
feature_cols = [c for c in df_reduced.columns if c not in ['timestamp', 'subject_id']]
X_scaled = RobustScaler().fit_transform(df_reduced[feature_cols].values)
X_pca = PCA(n_components=0.95, random_state=42).fit_transform(X_scaled)
# k=6 по min inertia среди k in [3,4,5,6]
labels = KMeans(n_clusters=6, random_state=42, n_init=10).fit_predict(X_pca)
# remap → 1..6, Index 0..534600
```

**Проверка воспроизведения:** inertia k=6 = 91473324.45, PCA shape (534601, 12).

## 6. Уроки для дальнейшей работы

1. **Silhouette / internal score ≠ Kaggle accuracy** — подтверждено многократно.
2. **Не менять scaler subject-wise** для submission без проверки LB.
3. **Не усложнять** без меток: v0 row-level PCA+KMeans k=6 остаётся лучшим известным решением.
4. План v1 (segment, ICA, grid) полезен для **отчёта/учёбы**, но не для LB без labeled validation.
5. Любое «улучшение» проверять **только отправкой на Kaggle**, не по proxy-метрикам.
