# Роли и навыки проекта «Кластеризация физической активности»

Проект: безнадзорная кластеризация типов физической активности по данным IMU (рука, грудь, лодыжка) и пульсометра; сдача — воспроизводимый ноутбук и submission на Kaggle.

---

## 1. Владелец решения / ML-инженер (кластеризация)

**Зона ответственности:** постановка пайплайна обучения, выбор и сравнение алгоритмов кластеризации, подбор гиперпараметров, формирование submission, обоснование финальной модели.

**Навыки:**
- unsupervised learning: K-Means, DBSCAN, Agglomerative Clustering, GMM, при необходимости — spectral / HDBSCAN;
- метрики без меток: silhouette, Davies–Bouldin, Calinski–Harabasz; понимание ограничений метрик на высокоразмерных данных;
- масштабирование признаков, снижение размерности (PCA, UMAP — с обоснованием);
- подбор числа кластеров: elbow, silhouette, gap statistic;
- воспроизводимость: `random_state`, фиксация версий библиотек;
- постобработка меток кластеров под формат Kaggle (`Index`, `activityID`, упорядочивание ID).

---

## 2. Аналитик данных (EDA и предобработка)

**Зона ответственности:** разведочный анализ, очистка данных, выявление выбросов и пропусков, обоснованная предобработка, выводы для моделирования.

**Навыки:**
- pandas, NumPy; работа с табличными временными рядами (54 признака);
- описательная статистика, корреляционный анализ, мультиколлинеарность;
- визуализация: распределения, pairplot/heatmap, временные профили по сенсорам и ЧСС;
- обработка пропусков и аномалий; фильтрация нерелевантных признаков;
- структурирование EDA в ноутбуке с выводами к каждому этапу (критерий 1 задания).

---

## 3. Инженер признаков (сенсорные данные / HAR)

**Зона ответственности:** feature engineering для IMU и пульса; агрегация и извлечение информативных характеристик из сырых измерений.

**Навыки:**
- домен PAMAP2 / Human Activity Recognition: семантика аксelerometer, gyroscope, magnetometer, температуры, ЧСС;
- статистические агрегаты по окнам: mean, std, min, max, RMS, energy, zero-crossing rate;
- спектральные и временные признаки (FFT, домinant frequency — при необходимости);
- нормализация по сенсору и по телу (рука / грудь / лодыжка);
- оценка важности признаков для интерпретации кластеров (variance, PCA loadings, профили центроидов).

---

## 4. Специалист по интерпретации и валидации результатов

**Зона ответственности:** содержательная интерпретация кластеров, связь с типами активности, анализ ошибок и направления улучшения.

**Навыки:**
- профилирование кластеров по признакам (centroid analysis, radar/box plots по группам);
- сопоставление кластеров с ожидаемыми классами активности (ходьба, бег, велосипед и др.) без использования меток при обучении;
- анализ устойчивости: bootstrap, повторные запуски, чувствительность к гиперпараметрам;
- формулировка выводов и рекомендаций (критерий 3 задания).

---

## 5. Разработчик ноутбука (Python / оформление решения)

**Зона ответственности:** структура `.ipynb`, читаемый код, комментарии, воспроизводимый end-to-end пайплайн, артеfact submission и скриншот лидерборда.

**Навыки:**
- Jupyter; модульная организация: EDA → preprocessing → features → clustering → evaluation → predict;
- PEP 8, функции для повторяющихся операций, отсутствие дублирования;
- экспорт `submission.csv` в формате `Index,activityID`;
- фиксация окружения (`requirements.txt` / `%pip freeze` — по согласованию);
- документирование этапов и выводов (критерии 1, 2, 4 задания).

---

## 6. Интегратор Kaggle / QA

**Зона ответственности:** участие в соревновании, проверка формата ответа, контроль воспроизводимости и полноты сдачи.

**Навыки:**
- работа с Kaggle API / веб-интерфейсом, правила соревнования;
- валидация submission: заголовок, типы, непрерывная нумерация `activityID`;
- smoke-тест пайплайна «с нуля» на чистом окружении;
- контроль чек-листа сдачи: ноутбук, скриншот leaderboard, платформа курса.
